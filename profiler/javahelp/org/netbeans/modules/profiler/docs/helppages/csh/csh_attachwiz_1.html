Attach Wizard: Select Target Type

See Also

The Attach Wizard helps you configure the settings for profiling an application using the attach method. In most cases you only need to use the Attach wizard to profile an application running on a remote JVM. If your application is running on a local JVM and you can start the application from the IDE, you can use Profile Main Project. For more information on when to use the attach mode, see Profiling Using Attach Mode.

You use this page of the Attach wizard to specify the type of application you want to profile and how you want to attach to the application. After specifying the attach settings, click Next to review the attach settings.

In this page of the wizard you specify the following settings:

  • Target Type. Select the type of project that you want to profile. You can select from following types:
    • Application. Select this if your project is a standalone J2SE application.
    • Applet. Select this if your project is an applet.
    • Java EE Web/App Server. Select this if you are profiling an enterprise application or web application deployed to a web or application server.

    If you select Java EE Web/App Server, you also need to specify the following:

    • Java EE Web/Application Server Type. Select the deployment server for the application
  • Attach method. Select an attach method according to the machine where the application is running.
    • Local. Select Local if your application is running on the same machine as the IDE.
    • Remote. Select Remote if the application is running on a remote machine.
    • If you choose the Remote attach method, you need to install the Profiler Remote Pack on the target machine. You can generate the Profiler Remote Pack using the Attach Wizard.
  • Attach invocation. Select the type of attach method.
    • Direct. Select Direct if you want to profile application startup. When you choose this option, the application startup is paused until the IDE is attached to the application/JVM.
    • Dynamic. Select Dynamic if you want to attach and reattach the IDE. When you choose this option, the application is started normally. You can then attach, detach and reattach to the application without restarting the application.

    Notes:

    • The Dynamic attach method requires that the target application is running on JDK 1.6.
    • The Dynamic attach method is not available if you are profiling a remote application.
